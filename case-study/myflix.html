Case Study for MyFlix Project

Previous Next

Overview: I created this movie index project to learn the methods to create a full front- and back-end project from scratch using the MERN stack. I really enjoyed the project because it really let me dig into React which has proven to be one of my favorite frameworks to work with, and the subject matter was something that I’m very passionate about. Movies are my wife and I’s go-to date night when we get the chance, and I created the database using my some of my favorite movies from the Marvel Cinematic Universe. The project was started by creating a database using MongoDB, and API using Node/Express which was uploaded and hosted on Heroku. The client for the app was created using React and was meant to discover the uses of the framework and how to manipulate the virtual DOM, states, and props.

To begin the project, I created the API endpoints that were necessary to access the database that was separated into users and movies, and identified the REST functionality for the user access. Once I had the databases and endpoints setup, I tested the API using Postman to make sure that everything was working properly. After testing, I added authentication and authorization using Passport, and uploaded my API to Heroku.

After the API was taken care of, I created the React client using user flows and wire frames to detail the layout of the single page web app. After the layout was created, I implemented the necessary components and functions, created a simple style with React Bootstrap, and hosted the app on Heroku with the API. The app is live at https://myflixdb5253.herokuapp.com/client/.

The biggest challenge I faced with this project was an issue with the CORS policy while the client was hosted on Heroku. The problem though, was that my error codes weren’t specific enough, so I spent a lot of time combing through the code to figure out where the issue was located. Now I have a better understanding of implement try/catch statements and have more experience with debugging my code. Luckily, the project was completed in a little under 2 weeks, despite a delay with the error, and I was able to turn the project over in my desired time frame.
